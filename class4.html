Dealing with a merge conflict

I've set up the Moby Dick project on GitHub. It has two branches, and I have set up the files that they will give you a merge conflict when you try to merge them to give you practice. Try to clone and bring down the additional branch on your own, but if you need help, see the instructions below:

Bring down the branches

  1. Clone this repository: https://github.com/katebron/Moby_dick
  2. Run git branch -a to see all the branches
  3. To automatically set up a new branch and pull my additional GitHub branch into it, run
    4. git checkout another_moby
  4. Run git diff to see the differences betwen the two branches

Merge the branches

In this exercise, I'd like you to merge another_moby into master. Try to do this part on your own - feel free to Google. I will have 'solutions' next week.

  • Move to master
  • Merge another_moby into master
  • Deal with the merge conflict
