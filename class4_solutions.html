Dealing with a merge conflict

Merge the branches

In this exercise, I'd like you to merge the change_index branchinto master. Try to do this part on your own - feel free to Google. I will have 'solutions' next week.

  • Move to master
    • git checkout master
  • Merge change_index into master
    • git merge change_index
  • Deal with the merge conflict

    • Open index.html with a text editor.
    • Look for the "<<<<<<< HEAD" and "=====" notations: This is git's way of pointing out the exact area(s) where the conflicts are.
    • Decide which line you want to take. Erase the other one, as well as all the "<<<..", "===" and branch name notations.
    • Add & commit this file.
      git add .
      git commit -m "something about merging the change_index branch"
